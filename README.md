# Network Intrusion Detection with Stacked ML Models

A network intrusion detection system (NIDS) built on the **UNSW-NB15** dataset that combines five base classifiers into a stacking ensemble to detect normal traffic and 9 categories of network attacks. Includes a Flask web app for uploading traffic CSVs and visualizing predictions in real time, with optional hardware integration (Arduino / NodeMCU) for physical alerts.

**Author:** Abishek Udayakrishna Surya Narayanan
**Project Window:** Jul 2024 – Dec 2024

---

## Overview

The goal of this project is to classify network flow records as either normal traffic or one of several attack categories — including DoS, reconnaissance, exploits, fuzzers, worms, backdoors, analysis, and generic attacks — and demonstrate that a stacking ensemble outperforms each of its constituent base learners.

The system has three layers:

1. **Training pipeline** (`multiclass.py`) — Preprocesses UNSW-NB15, trains 5 base classifiers and a stacking meta-classifier, saves the model.
2. **Inference web app** (`app.py`) — Flask app that accepts a CSV of flow records, runs predictions in a background thread, and streams results to the browser.
3. **Hardware alert layer** (`serial_test.py`) — Pushes encoded attack labels over a serial connection to an Arduino / NodeMCU for physical alerts (LCD display, LED indication).

---

## Dataset

**UNSW-NB15** ([UNSW Canberra ADFA](https://research.unsw.edu.au/projects/unsw-nb15-dataset)) — a modern intrusion detection benchmark generated by the IXIA PerfectStorm tool with a hybrid of real normal traffic and synthetic contemporary attacks.

| Property | Value |
| --- | --- |
| Features | 49 (mix of nominal, integer, binary, float) |
| Attack categories | 9 (Normal, Generic, Exploits, Fuzzers, DoS, Reconnaissance, Analysis, Backdoor, Worms) |
| File used | `UNSW_NB15.csv` (with `UNSW_NB15_features.csv` for type metadata) |

## Pipeline (multiclass.py)

```
Load CSV
    │
    ▼
Drop rows where service == '-' (treated as missing)
    │
    ▼
Type-cast nominal/integer/binary/float columns per features file
    │
    ▼
One-hot encode categorical attributes (proto, service, state, ...)
    │
    ▼
MinMax normalize all numeric columns to [0, 1]
    │
    ▼
Label encode:
   • binary labels (normal / abnormal)  →  le1_classes.npy
   • multi-class labels (9 categories)  →  le2_classes.npy
    │
    ▼
Train/test split (80/20, random_state=100)
    │
    ▼
Train 5 base classifiers individually + Stacking Ensemble
    │
    ▼
Save final model → models/stacking_classifier.pkl
```

### Models trained

| Model | Configuration |
| --- | --- |
| AdaBoost | `n_estimators=100, random_state=123` |
| Gradient Boosting | `n_estimators=100, random_state=123` |
| Quadratic Discriminant Analysis | sklearn defaults |
| Linear SVM (One-vs-Rest) | `kernel='linear', decision_function_shape='ovr', probability=True` |
| Bagging | `n_estimators=50, random_state=123` |
| **Stacking** | 5 base learners → Logistic Regression meta-classifier (`solver='newton-cg', max_iter=5000, multinomial`) |

Per-model classification reports and Real-vs-Predicted plots are saved to the `plots/` folder.

## Web Application (app.py)

A Flask app served on `http://localhost:9000` that wraps the trained model for live inference.

### Flow

1. User uploads a CSV of network flow records via the browser.
2. App extracts the 15 features the deployed model uses (a reduced subset of UNSW-NB15):
   - `dttl, swin, dwin, tcprtt, synack, ackdat`
   - `proto_tcp, proto_udp, service_dns`
   - `state_CON, state_FIN`
   - `ct_flw_http_mthd, ct_src_ltm, ct_srv_dst, is_sm_ips_ports`
3. Features are MinMax-scaled, then passed to `model_.pkl` (the deployed model used by the web app).
4. Predictions stream to the frontend via `/data` (polled JSON endpoint).
5. `/bargraph` renders a real-time bar chart of attack-type distribution.
6. `/stop` interrupts the background prediction thread.

### Endpoints

| Route | Method | Purpose |
| --- | --- | --- |
| `/` | GET | Render upload page |
| `/upload` | POST | Accept CSV and start background prediction |
| `/data` | GET | Return current input and predicted data as JSON |
| `/bargraph` | GET | Return PNG of attack-type frequency |
| `/stop` | POST | Signal background thread to stop |

## Hardware Integration

The serial code path in `app.py` (currently commented out) and `serial_test.py` push predicted labels over `COM10` at 9600 baud to an Arduino / NodeMCU board. Labels are mapped to single-character codes for efficient serial transmission:

```
Generic → a    Worms → b    Analysis → c
Fuzzer  → d    Exploits → e Normal   → f
Backdoor→ g    Reconnaissance → h     DoS → i
```

The microcontroller decodes the character, displays the corresponding attack type on an attached LCD, and can trigger LEDs / buzzers for alerts.

> The serial block is commented out by default so the app runs without hardware present. To enable, uncomment the `ser = serial.Serial(...)` lines in `app.py` and adjust `COM10` to your port.

## Setup

### Requirements

- Python 3.10+
- scikit-learn, pandas, numpy, matplotlib, seaborn
- Flask, joblib
- pyserial (only if using hardware alerts)
- Arduino IDE + Arduino / NodeMCU board (optional, for hardware integration)

### Installation

```bash
git clone https://github.com/abishek1310/intrusion-project.git
cd intrusion-project

# (Recommended) Create a virtual environment
python -m venv venv
source venv/bin/activate      # On Windows: venv\Scripts\activate

pip install scikit-learn pandas numpy matplotlib seaborn flask joblib pyserial
```

### Train the models from scratch

```bash
python multiclass.py
```

This will train all 5 base classifiers + stacking, save plots to `plots/`, and write the final model to `models/stacking_classifier.pkl`.

### Run the web app

```bash
python app.py
# Open http://localhost:9000 in your browser
```

Upload a CSV with the 15 features listed above, then watch predictions appear in real time.
